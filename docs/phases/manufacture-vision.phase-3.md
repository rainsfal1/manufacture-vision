## What to build

Add the evidence capture layer to the perception node. The ingestion reader is split into a dual-stream: the low-res substream feeds the existing AI inference queue, while the high-res mainstream feeds a ring buffer that retains the last N seconds of frames. When an event fires, the clip exporter slices a `[t-5s, t+5s]` window from the ring buffer and writes it to a temporary file. The S3 uploader sends it asynchronously to MinIO and attaches a `clip_ref` to the event payload before it publishes to Redis. If MinIO is unreachable, the event publishes without `clip_ref` and the upload is retried in the background. MinIO runs in Docker alongside the edge node.

## Architectural decisions

- Dual-stream split happens at the reader level — low-res (e.g. 640px) for inference speed, high-res (native) for clip quality.
- Ring buffer size is configurable (default: 30 seconds of high-res frames).
- Clip window: `t-5s` to `t+5s` relative to event `event_ts_ms`.
- `clip_ref` schema: `{ bucket, key, expires_at }` — S3-compatible, MinIO-backed.
- Upload is async and non-blocking — evidence capture must never slow the inference loop.
- MinIO bucket policy: private. All access via signed URLs (generated by central backend in Phase 4).
- If upload fails after retries: event is permanently marked `clip_ref: null` in the log.

## Acceptance criteria

- [ ] Dual-stream split delivers low-res frames to inference queue and high-res to ring buffer concurrently.
- [ ] Ring buffer retains ≥30 seconds of high-res frames without memory runaway.
- [ ] Clip exporter produces a valid MP4 covering `[t-5s, t+5s]` for every triggered event.
- [ ] S3 uploader sends clip to MinIO and attaches `clip_ref` to the event before Redis publish.
- [ ] MinIO outage: event publishes without `clip_ref`; upload retried; inference continues uninterrupted.
- [ ] MinIO runs in Docker and is accessible from the edge node container.
- [ ] Telemetry includes clip upload success/failure counter.

## Notes

Signed URL generation is NOT in this phase — that belongs to the central backend (Phase 4). Clips are in MinIO but not yet accessible to the dashboard until Phase 5. This phase makes the evidence exist; Phase 5 makes it viewable.
